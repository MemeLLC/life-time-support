# 市場調査（Market Analysis）

対象市場の市場規模・市場構造・価格帯・顧客像を調査し、レポートにまとめる。

## ワークフロー

```
Phase 1: ヒアリング → Phase 2: 調査実施 → Phase 3: レポート化
```

---

### Phase 1: ヒアリング

調査に着手する前に、ユーザーに質問して方向性を定める。

- **調査対象**: どの市場・プロダクトカテゴリを調べるか？
- **目的・背景**: なぜ調べるのか？（新規参入の検討 / 既存事業の理解深化 / 投資判断 / 提案資料の準備 等）
- **市場の地理的範囲**: 国内市場が基本。ただし海外市場が国内ビジネスに影響する場合（輸入品、グローバルプラットフォーム、海外トレンドの波及等）は海外データも調査対象に含める
- **B2B / B2C**: ビジネスモデルによって顧客調査の切り口が変わる（B2B: 企業規模・業種・意思決定プロセス / B2C: 年齢・性別・年収・ライフスタイル）
- **重視する領域**: 4つの調査領域のうち、特に重点を置きたい領域はあるか？（なければ全領域を実施）

#### 前提条件

`industry-landscape` スキルで対象業界の前提知識（用語・構造・ビジネスモデル）が把握済みであることが望ましい。未実施の場合は先に実行を提案する。

### Phase 2: 調査実施

以下の4領域を調査する。各領域の具体的な調査項目・クエリ例は [references/research-guide.md](references/research-guide.md) を参照すること。

1. **市場規模** — 金額・数量ベースの市場規模、成長率、将来予測、セグメント別内訳
2. **市場構造** — 主要プレイヤーとシェア、流通チャネル、参入障壁、市場の成熟度
3. **価格調査** — 市場の価格帯、料金モデル、価格戦略の傾向
4. **顧客調査** — ターゲット層の属性データ、購買行動、ペルソナ像

#### 地理的範囲の扱い

- **国内完結ビジネス**: 日本市場のデータに集中する。クエリは日本語中心
- **海外市場が影響するビジネス**: 国内データを軸としつつ、グローバル市場規模・海外の価格水準・海外トレンドも調査する。クエリは日本語＋英語を併用

### 検索ルール

#### 複数ソースの原則

1つのトピックにつき**最低3〜5回**の異なるクエリで検索し、**2〜3件以上**のソースをWebFetchで本文まで読むこと。特に市場規模データは最低3つのソースで突き合わせる。

#### クエリ設計

- **広→狭のファネル戦略**: まず全体像を掴み、次に絞り込む
- **毎回クエリを変える**: 類義語・視点を変える
- **年を入れる**: 鮮度が重要な情報には必ず年を明記
- **言語はビジネスに合わせる**: 国内市場は日本語中心、海外データが必要な場合は英語も併用

#### 検索演算子の活用

| 演算子 | 用途 | 例 |
|--------|------|-----|
| `"完全一致"` | フレーズ完全一致検索 | `"市場規模 2025" SaaS` |
| `site:` | 特定サイト内検索 | `site:meti.go.jp 電子商取引` |
| `filetype:` | ファイル形式指定 | `filetype:pdf [業種] 市場調査 2025` |
| `intitle:` | タイトル内検索 | `intitle:市場規模 [業種]` |

**一次情報を狙うsite:指定:**
- 官公庁: `site:meti.go.jp` `site:stat.go.jp`
- 調査会社: `site:statista.com` `site:grandviewresearch.com`
- IR情報: `site:jpx.co.jp`

#### 情報収集ツール

Web検索（WebSearch）とWebFetchに加え、`playwright-browser` スキルのscrapeコマンドを活用してページの詳細情報を取得すること。特に調査会社のレポートページや官公庁の統計ページなど、構造的なデータが含まれるページの取得に有効。

```
Step 1: WebSearchで検索 → URLリストを取得
Step 2: WebFetchまたはplaywright-browserのscrapeで本文を取得
```

スニペットだけで判断せず、重要な情報はWebFetchで本文を読むこと。

### Phase 3: レポート化

調査結果を [references/output-template.md](references/output-template.md) のテンプレートに沿って整理し、マークダウンファイルとして出力する。出力先はユーザーの指示に従う。

後続のリサーチスキルへの接続を案内する（関連スキルとの連携セクションを参照）。

## アウトプットルール

### 網羅的な記述

- 数値データは具体的な数字を記載する
- 複数ソースから得た情報はすべて記載し、差異があればそれも明記する
- 「など」「その他多数」で省略せず、把握した項目はすべて列挙する

### ソースURLの記載

すべての発見事項・データには、情報源のURLを必ず記載すること。URLのないデータは出力に含めない。

## 関連スキルとの連携

| スキル | 連携内容 |
|--------|----------|
| **industry-landscape** | 業界の前提知識（用語・構造・ビジネスモデル）が揃っていると、市場調査の検索精度が上がる。industry-landscape → market-analysis の順が望ましい |
| **competitor-analysis** | 市場構造・シェアの理解があると、競合分析でのポジショニング評価が正確になる。market-analysis → competitor-analysis の順が望ましい |
| **regulatory-research** | 市場の参入障壁に規制要因がある場合、法規制調査に接続する |
| **business-profile** | 自社の事業内容が整理済みであれば、市場内での自社の位置づけを意識した調査ができる |
| **playwright-browser** | 調査会社のレポートページ、官公庁の統計ページなどの情報取得に使用する |
