# Google Ads 戦略・設計スキル

Google Adsの広告タイプ選定、キャンペーン設計、運用仕様書の作成を一貫して支援する。

## ワークフロー

```
Step 0: モード判定       → 新規立ち上げ or 既存運用の改善 を判定
    ↓
Step 1: 情報収集（Phase A）→ ビジネスの基本情報を把握し、広告タイプの方向性を提示
    ↓
Step 1: 情報収集（Phase B）→ 設計に必要な詳細情報を収集
    ↓
Step 2: 広告タイプ選定    → 状況に最適な広告タイプと構成を決める
    ↓
Step 3: 戦略立案          → キャンペーン構造・入札・ターゲティングの方針を決める
    ↓
Step 4: 仕様書作成        → 各広告タイプのプレイブックに基づいて具体的な設計を行い、仕様書を出力する
```

---

## Step 0: モード判定

ユーザーの状況から以下のどちらかを判定する。明確でなければ質問する。

| モード | 条件 | ワークフロー |
|--------|------|-------------|
| **新規立ち上げ** | Google Adsを未運用 or 新キャンペーンをゼロから設計 | Step 1 → 2 → 3 → 4 の順に進む |
| **既存運用の改善** | Google Adsを運用中で、課題がある or パフォーマンスを改善したい | Step 1 → 診断 → 改善提案 → 必要に応じて Step 3〜4 |

### 既存運用の改善フロー

既存運用の場合、Step 1 の後に以下の診断プロセスを行う:

1. **現状把握**: アカウント構造、キャンペーン一覧、現在のKPI実績を確認
2. **課題特定**: ユーザーの課題感をヒアリングし、以下の観点で分析
   - 入札戦略は適切か（CVデータ量に対して高度すぎる入札を使っていないか）
   - アカウント構造に問題はないか（キャンペーンの細分化しすぎ、広告グループのテーマ混在等）
   - 除外KWは十分か、検索語句レポートで無駄なクリックが発生していないか
   - CVトラッキングは正しく設定されているか
   - 品質スコアが低い広告グループはないか
   - 予算の配分は適切か（高パフォーマンスCPに予算が足りていない等）
3. **改善提案**: 優先度順に改善アクションを提示し、ユーザーと合意
4. **実行**: 合意した改善を仕様書として出力（Step 4 のテンプレートを活用）

## Step 1: 情報収集

### Phase A: 基本情報（広告タイプの方向性を決めるのに必要な最低限の情報）

| カテゴリ | 確認事項 |
|---------|---------|
| **ビジネスモデル** | EC / リード獲得 / アプリ / 店舗集客 / ブランド認知のどれか |
| **商材** | 何を売る/訴求するか（商品、サービス、無料トライアル、資料DLなど） |
| **目標** | 主要KPIは何か（CPA、ROAS、CPI、CPV等）。目標値があれば数値も |

Phase A の回答を得たら、Step 2 の早見表をもとに **広告タイプの方向性を提示** する。ユーザーが方向性に納得したら Phase B に進む。

### Phase B: 設計に必要な詳細情報

| カテゴリ | 確認事項 |
|---------|---------|
| **予算** | 月次 or 日次の広告予算 |
| **LP** | 遷移先のランディングページURL |
| **ターゲット** | 誰に届けたいか（地域、年齢層、業種、検索行動など） |
| **既存運用** | 現在Google Adsを運用中か。運用中なら課題は何か |
| **CVデータ** | 過去30日のCV件数（入札戦略の選定に影響） |
| **商品フィード** | Merchant Centerの商品フィードはあるか（EC向け） |
| **クリエイティブ** | 動画・画像素材はあるか、制作体制はあるか |
| **計測** | コンバージョントラッキングは設定済みか |

`contexts/` にビジネスプロファイルや競合分析がある場合はそこから取得し、既知の情報は質問せずスキップする。

## Step 2: 広告タイプ選定

### キャンペーンタイプ早見表

| キャンペーン | 配信面 | 課金 | ファネル | 自動化度 | 適した業種 |
|---|---|---|---|---|---|
| **検索** | Google検索 | CPC | 下部（顕在層） | 中 | 全業種 |
| **ディスプレイ** | GDN（200万+サイト） | CPC/CPM | 上部（認知） | 中 | 全業種 |
| **ショッピング** | 検索/ショッピングタブ | CPC | 下部（購買） | 中 | EC・物販 |
| **動画** | YouTube/動画パートナー | CPV/CPM | 上部〜中部 | 中 | 全業種 |
| **アプリ** | 検索/Play/YouTube/GDN | CPI/CPA | 中部〜下部 | 高 | アプリ事業者 |
| **P-MAX** | 全7チャネル | CPA/ROAS | 全段階 | 非常に高 | 全業種 |
| **Demand Gen** | YouTube/Discover/Gmail/GDN | CPC/CPA/CPM | 上部〜中部 | 高 | 全業種 |

### 各キャンペーンタイプの特徴

**検索キャンペーン**
- Google検索結果にテキスト広告を表示。購買意欲の高い顕在層へのリーチに最適
- RSA（レスポンシブ検索広告）: 見出し最大15個+説明文4個をAIが最適組み合わせ
- AI Max for Search（2025年〜）: ブロードマッチ+自動生成+LP自動選択でCV14%向上
- 適: リード獲得、BtoB、EC特定商品、顕在層の刈り取り

**ディスプレイキャンペーン**
- GDN 200万以上のサイト・アプリに画像/動画広告を配信。ネットユーザー約90%にリーチ可能
- RDA（レスポンシブディスプレイ広告）: 画像+テキストをAIが組み合わせて配信面に自動調整
- ターゲティング: オーディエンス（興味関心/カスタム/類似）、コンテンツ（トピック/プレースメント）、リマーケティング
- 適: ブランド認知、リマーケティング、潜在層アプローチ

**ショッピングキャンペーン**
- 商品画像・価格・店舗名を含むリスト形式で検索結果に表示。Merchant Centerのフィード必須
- Standard Shopping: 手動入札で細かい制御可能
- P-MAXショッピング: AI自動最適化で全チャネルに配信（旧スマートショッピング）
- 適: ECサイト、オンラインストア、価格競争力のある商品

**動画キャンペーン（YouTube広告）**
- YouTube/動画パートナーに動画広告を配信
- フォーマット: スキッパブル/ノンスキッパブル/バンパー（6秒）/インフィード/マストヘッド/アウトストリーム
- 2025年変更: VAC→Demand Genに統合完了、Shorts縦型広告が全タイプ対応
- 適: ブランド認知、新商品ローンチ、ハウツー訴求

**アプリキャンペーン**
- アプリのインストール/エンゲージメント/事前登録に特化。全プラットフォームに自動配信
- ACi（インストール）/ ACe（エンゲージメント）/ ACpre（事前登録）
- KW指定・プレースメント指定は不可（全自動）
- 適: アプリのユーザー獲得、再エンゲージメント

**P-MAX（Performance Max）**
- 1キャンペーンで検索/ショッピング/ディスプレイ/YouTube/Gmail/Discover/マップの全7チャネルに配信
- アセットグループ+オーディエンスシグナル+検索テーマをAIが自動最適化
- 2025年: 除外KW 10,000件対応、チャネル別レポート、デモグラフィック除外、A/Bテスト
- 適: EC売上最大化、リード獲得最大化、店舗集客、リソースが限られたチーム

**Demand Gen（デマンドジェネレーション）**
- YouTube/Discover/Gmail/GDNにビジュアルリッチな広告を配信。Meta広告のGoogle版
- 画像/カルーセル/動画/商品フィード付き広告に対応
- 2025-2026年: GDN追加、Target CPC、新規顧客獲得モード、配信面個別選択
- 適: ミッドファネル、SNS広告的なビジュアル訴求、YouTube Shorts

### ビジネスモデル別 推奨構成

| ビジネスモデル | 第一候補 | 第二候補 | 条件・補足 |
|--------------|---------|---------|-----------|
| **EC（商品販売）** | P-MAX（フィード連携） | 検索（高意図KW）+ ショッピング | 商品フィード必須。月CV 30件以上で P-MAX が有利 |
| **リード獲得（BtoB）** | 検索広告 | P-MAX（フルアセット） | 検索意図が明確。P-MAXはURL拡張OFFで使う |
| **リード獲得（BtoC）** | 検索 + P-MAX | ディスプレイ（リマーケティング） | 検索で刈り取り、P-MAXで面を広げる |
| **アプリ** | アプリキャンペーン | 動画（YouTube） | ACi/ACe/ACpre から目的で選択 |
| **ブランド認知** | 動画（YouTube） | ディスプレイ / Demand Gen | 動画素材の有無で判断 |
| **店舗集客** | P-MAX（Store Goals） | 検索（地域KW） | Googleビジネスプロフィール連携必須 |

### 予算別の推奨

| 月予算 | 推奨構成 |
|-------|---------|
| ~10万円 | 検索広告1本に集中（ブランドKW + 高意図KW） |
| 10~30万円 | 検索 + P-MAX or ディスプレイ（リマケ） |
| 30~100万円 | 検索 + P-MAX + 動画 or ディスプレイ |
| 100万円~ | 全チャネル活用。キャンペーン目的別に分離 |

### Google推奨「パワーパック」構成（2025-2026年）

1. **Performance Max** — コンバージョン最大化のメインエンジン
2. **Demand Gen** — ビジュアルリッチな潜在層開拓
3. **AI Max for Search** — 検索キャンペーンのAI強化

### 選定後のアクション

選定したら以下を提示してユーザーに方向性の確認を取る:
1. 推奨する広告タイプとその理由
2. キャンペーン構成の概要（何本のキャンペーン、各キャンペーンの役割）
3. 各キャンペーンの入札戦略
4. 予算配分

## Step 3: 戦略立案

このステップではキャンペーンタイプ横断の戦略方針を決める。各広告タイプ固有の詳細設計は Step 4 で各プレイブックを参照して行う。

### アカウント構造

Google Adsは3層構造: アカウント → キャンペーン → 広告グループ（→ 広告 + キーワード）

| 階層 | 設計原則 |
|------|---------|
| キャンペーン | 目的別・予算別に分離。推奨6〜10本。細分化しすぎるとAI学習データが分散する |
| 広告グループ | 1グループ=1テーマ。KW 10〜20個、RSA 3〜5個。1キャンペーンあたり7〜10グループ |

### 命名規則

アカウント内のキャンペーン・広告グループ・アセットグループの命名を統一する。命名規則は運用効率に直結する。フィルタ・レポート・一覧性のすべてに影響するため、初期設計段階で必ず定めること。

#### キャンペーン名

**フォーマット:** `{タイプ}_{目的}_{ターゲット}_{地域}_{補足}`

| 要素 | 設定値 | 説明 |
|------|--------|------|
| タイプ | `Search` `PMax` `Display` `Shopping` `Video` `DGen` `App` | キャンペーンタイプの略称 |
| 目的 | `CV` `Lead` `Sales` `Awareness` `Traffic` `Install` | キャンペーンの主目的 |
| ターゲット | `Brand` `NonBrand` `Competitor` `Remarketing` `Prospecting` `AllProducts` | 対象セグメント |
| 地域 | `JP` `Tokyo` `Kansai` 等 | 地域ターゲティング（全国の場合は省略可） |
| 補足 | 自由記述（商品カテゴリ、テスト名など） | 必要に応じて |

**例:**

| キャンペーン | 名前例 |
|------------|--------|
| ブランド検索 | `Search_CV_Brand_JP` |
| 非ブランド検索 | `Search_Lead_NonBrand_Tokyo` |
| P-MAX（EC全商品） | `PMax_Sales_AllProducts` |
| P-MAX（カテゴリ別） | `PMax_Sales_Shoes` |
| ディスプレイ（リマケ） | `Display_CV_Remarketing` |
| ディスプレイ（認知） | `Display_Awareness_Prospecting` |
| Demand Gen | `DGen_Lead_Prospecting` |
| 動画（認知） | `Video_Awareness_YouTube` |
| ショッピング | `Shopping_Sales_Electronics` |
| アプリ | `App_Install_iOS` |

#### 広告グループ / アセットグループ名

**フォーマット:** `{テーマ}_{サブ分類}`

| キャンペーンタイプ | テーマ例 | サブ分類例 |
|-----------------|---------|-----------|
| 検索 | KWカテゴリ（`CRM` `導入費用` `比較`） | マッチタイプ（`Exact` `Phrase` `Broad`） |
| P-MAX | ターゲットセグメント（`NewCustomer` `Retarget`） | 商品カテゴリや訴求 |
| ディスプレイ | ターゲティング手法（`Interest` `Custom` `Placement`） | 具体的なセグメント名 |
| Demand Gen | 配信面・目的（`YouTube` `Discover` `Carousel`） | オーディエンス名 |
| ショッピング | 商品カテゴリ（`Shoes` `Bags`） | 価格帯やブランド |

**例:** `CRM_Exact` / `導入費用_Phrase` / `NewCustomer_HighValue` / `Interest_ITManager`

#### 命名ルール

- **区切り文字はアンダースコア `_` で統一**。ハイフンやスペースを混在させない
- **英語ベース推奨**。Google Ads管理画面のフィルタや外部ツールとの互換性が高い
- **日付はキャンペーン名に含めない**。Google Adsが開始日を記録するため冗長。テスト施策のみ `_Test_YYMM` を末尾に付与
- **大文字始まりのキャメルケース**（`NonBrand`, `AllProducts`）で単語を区切る
- **略称は統一リストを作る**。アカウント内で表記ゆれを防ぐ

### 入札戦略（フェーズ別）

| Phase | 条件 | 推奨戦略 |
|-------|------|---------|
| 1. 立ち上げ | CVデータなし | 手動CPC or クリック数の最大化 |
| 2. データ蓄積 | CVトラッキング済み | コンバージョン数の最大化 |
| 3. 最適化 | 30日で30件以上のCV | 目標CPA（過去平均CPAから開始） |
| 4. 拡大 | 50日で50件以上 + CV値あり | 目標ROAS / コンバージョン値の最大化 |

入札変更時の注意:
- 学習期間（2〜3週間）中は大幅変更を避ける
- 目標CPA/ROASは一度に20%以上変更しない
- 日予算は目標CPAの10〜15倍以上を確保

### コンバージョン設計

| 種別 | 用途 | 例 |
|------|------|-----|
| プライマリCV | 入札最適化に使用 | 購入、問い合わせ、有効リード |
| セカンダリCV | モニタリング用 | 資料DL、メルマガ登録 |
| マイクロCV | エンゲージメント計測 | スクロール深度、滞在時間 |

2025年〜の必須設定:
- **拡張コンバージョン（Enhanced Conversions）**: ファーストパーティデータをハッシュ化送信。Cookie制限対策として必須レベル
- **オフラインCV（リード獲得型）**: CRM成約データをGoogle Adsにフィードバック
- **サーバーサイドタギング**: アドブロッカー・Cookie制限を軽減

#### VTC（ビュースルーコンバージョン）の統一方針

VTCは広告を見たがクリックせず、後日CVしたユーザーを計測する指標。ディスプレイ、動画、P-MAX、Demand Gen など表示型の広告で発生する。広告タイプによって VTC の影響度が異なるため、以下の統一方針で運用する。

| 原則 | 説明 |
|------|------|
| **デフォルトはモニタリング** | VTC は入札最適化（Primary CV）に含めず、レポートで参考値として追跡する。これが最も安全な出発点 |
| **ウィンドウは短く** | VTC の計測ウィンドウは 1日（デフォルト）を維持。長くすると因果関係が薄まり過大計上になる |
| **合算しない** | VTC を直接 CV と合算して CPA/ROAS を評価しない。別枠で「間接貢献」として報告する |
| **VTC 比率を監視** | VTC が全 CV の 50% 以上を占める場合、実質的な広告効果が過大評価されている可能性が高い |

**広告タイプ別の VTC 対応:**

| 広告タイプ | VTC の影響度 | 推奨対応 |
|-----------|------------|---------|
| **検索** | 極小（検索はクリック中心） | 特別な対応不要 |
| **ディスプレイ** | 中〜高（表示量が多い） | VTC はセカンダリ CV として追跡。リマーケティングで CV 数が十分なら Primary に含めることも検討可 |
| **動画** | 中（視聴後の間接効果が大きい） | VTC は別枠で「動画の間接貢献」として評価。直接 CV と合算しない |
| **P-MAX** | 高（デフォルトで VTC 込みで最適化される） | VTC 比率を定期チェック。50% 超が続く場合はクリックベース CV アクションの作成を検討（上級者向け。詳細は [references/pmax.md](references/pmax.md) のセクション 9-3） |
| **Demand Gen** | 中〜高 | VTC はアシスト CV と合わせて「需要喚起の間接効果」として評価。ブレンド CPA で総合判断 |

### 予算配分

**70/20/10ルール:**
- 70%: 実績のあるキャンペーン・KW
- 20%: 入札・ターゲティング・広告文のテスト
- 10%: 新しいKW・オーディエンス・キャンペーンタイプ

### 運用サイクル

```
日次: 異常値チェック（急激なCPC上昇、CV急減）
週次: 検索語句レポート確認、除外KW追加、VTC比率チェック
隔週: 入札調整、広告文パフォーマンス確認
月次: KW追加・停止、予算配分見直し
四半期: キャンペーン構造見直し、戦略レベルの検討
```

### よくある失敗パターン

| 失敗 | 回避策 |
|------|--------|
| CVトラッキング不整合 | GTMプレビュー+GA4 DebugViewで検証 |
| 拡張CV未設定 | Cookie制限対策として設定必須 |
| 非現実的な入札目標 | 過去実績データから開始、段階的に改善 |
| 学習期間中の頻繁な変更 | 変更後2〜3週間は大幅変更を避ける |
| P-MAXの予算不足 | 日予算は目標CPAの10倍以上確保 |
| VTC込みのROASで判断 | クリックスルーCVのみのROASも併せて評価 |
| LPと広告のミスマッチ | 広告グループごとに対応LP用意 |

### 仕様書に含める項目

1. **戦略サマリー**: 目標、ターゲット、広告タイプ選定理由
2. **キャンペーン構成表**: 全キャンペーンの一覧（目的、入札、予算配分）
3. **広告タイプ別の設計詳細**: プレイブックに基づく具体的設計
4. **クリエイティブ制作指示**: 必要な素材の一覧と要件
5. **計測設計**: CV定義、VTC方針、トラッキング要件
6. **運用スケジュール**: 立ち上げ〜学習期間〜最適化の各フェーズ
7. **KPIと評価基準**: 成功の定義と測定方法

## 複数広告タイプの組み合わせパターン

### パターン1: 検索 + P-MAX（最も一般的）

```
検索CP（ブランドKW）  ← 自社ブランド防衛
検索CP（高意図KW）    ← 完全一致で刈り取り
P-MAX（フルアセット or フィード） ← 残りの面を自動最適化
```

検索で完全一致KWが優先される仕様を活用。P-MAXのブランド除外を設定しカニバリ防止。

### パターン2: EC フルファネル

```
P-MAX（フィード連携）           ← メイン。7チャネル自動配信
検索CP（非ブランド・高意図）     ← P-MAXが取りにくい指名検索系
Standard Shopping（オプション） ← P-MAXで成果が出ない商品群
ディスプレイ（リマケ）           ← カート落ち、閲覧者のリターゲ
```

### パターン3: リード獲得 BtoB

```
検索CP（指名KW + 課題KW）        ← 高意図ユーザー刈り取り
P-MAX（フルアセット、URL拡張OFF）← 認知拡大 + リマケ
動画CP（YouTube、オプション）     ← 専門性訴求、ブランド構築
```

### パターン4: ブランド認知 → コンバージョン

```
動画CP（YouTube Reach/Views）    ← 認知フェーズ
Demand Gen / ディスプレイ        ← 認知補完、ミッドファネル
検索CP（ブランドKW）              ← 認知後の指名検索を刈り取り
P-MAX                            ← 全面カバー
```
