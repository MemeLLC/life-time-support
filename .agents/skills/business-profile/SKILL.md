# ビジネスプロファイル（Business Profile）

自プロジェクトの情報を包括的に収集し、ドキュメントとしてまとめる。定量的な基本情報だけでなく、事業内容・ターゲット・ビジネスモデル・マーケティング施策・ECモール・アプリストア・口コミまで、可能な範囲でユーザーと協力しながら網羅する。

## ワークフロー

### Phase 1: ユーザーへのヒアリング

ユーザーに段階的に質問し、既知の情報を収集する。**一度にすべてを聞かず、3〜5項目ずつ段階的に質問すること。** ユーザーが「わからない」と答えた項目は Phase 2 で調査する。

#### Step 1: 基本情報（最初に聞く）

- 会社名（正式名称）と屋号・ブランド名
- サービス・プロジェクト名
- 何をしているビジネスか（一言で）
- ホームページ / サービスサイトのURL

#### Step 2: 会社の詳細

- 代表者名
- 従業員数（概算でOK）
- 設立年
- 所在地
- 資本金・資金調達状況（わかる範囲で）

#### Step 3: 事業内容・ターゲット

- 提供しているサービス・商品の詳細
- ターゲット顧客（B2B / B2C / B2B2C）
- 主な顧客層の特徴（業種、規模、年齢層など）
- 展開地域（日本全国 / 特定地域 / 海外）
- 現在の事業フェーズ（準備中 / ローンチ直後 / 成長期 / 安定期）
- 主な実績・導入事例（わかる範囲で）

#### Step 4: ビジネスモデル・料金

- 収益モデル（サブスク / 従量課金 / 買い切り / 広告 / フリーミアムなど）
- 料金体系・価格帯
- 主な収益源
- 他に展開している事業やサービスがあるか

#### Step 5: マーケティング・デジタルプレゼンス

- LP（ランディングページ）のURL
- SNSアカウント（X、Instagram、Facebook、LinkedIn、YouTube、TikTokなど）
- 現在出稿している広告の有無・プラットフォーム
- その他のマーケティング施策（SEO、コンテンツ、メルマガ、アフィリエイト、PR等）
- 現在の集客チャネルで最も効果的なもの

#### Step 6: ブランド・方向性（余裕があれば）

- ミッション・ビジョン
- ブランドの雰囲気・トーン（堅い / カジュアル / 高級感 / 親しみやすいなど）
- 現在の課題や悩み
- 短期的な目標（3〜6ヶ月）
- 中長期的な方向性

### Phase 2: Web調査による情報補完

ユーザーが把握していない情報、または裏取り・補完が必要な情報を調査する。Phase 1 のヒアリング結果をもとに、**どの調査を実施するかユーザーに確認してから着手する。**

#### 調査メニューの提示

以下の調査項目をユーザーに提示し、実施する範囲を決める。ビジネスの状況に応じて推奨セットを提案する。

| # | 調査項目 | 推奨する場面 |
|---|---------|------------|
| A | 会社・サービスの基本情報 | 常に（公式サイトがある場合） |
| B | ホームページ・LPの調査 | サイトURLが判明している場合 |
| C | SNS・外部プレゼンスの調査 | SNS運用中、または運用状況が不明な場合 |
| D | Google検索での可視性調査 | SEOが集客チャネルの場合、または現状把握したい場合 |
| E | サイトパフォーマンス調査（PSI） | サイトの速度・品質を把握したい場合 |
| F | 広告運用の調査 | 広告出稿中、または出稿状況が不明な場合 |
| G | ECモール調査（Amazon/楽天） | 物販・EC事業の場合 |
| H | アプリストア調査 | アプリ事業の場合 |
| I | 口コミ・レビューサイト調査 | レビューが購買意思決定に影響する業種の場合 |

**ビジネス状況別の推奨:**
- **新規事業（ローンチ前）**: A のみ（サイトやSNSがまだなければ調査対象がない）
- **ローンチ直後**: A + B + C
- **成長期〜安定期**: A〜F すべて推奨 + 該当する場合 G/H/I
- **物販・EC事業**: A〜F + G（Amazon/楽天に出品している場合）
- **アプリ事業**: A〜F + H
- **個人事業・フリーランス**: A + B + C（会社概要は簡略化）

#### A. 会社・サービスの基本情報

```
Step 1: WebSearchで「[会社名] 会社概要」「[会社名] 企業情報」等を検索
Step 2: WebFetchまたはplaywright-browserのscrapeで公式サイトの会社概要ページを取得
```

- 公式サイトの会社概要ページ、IR情報、プレスリリースを確認する
- 企業データベース（マイナビ、Wantedly、Crunchbase等）も補助的に参照する
- 代表者のプロフィール、メディア掲載、登壇情報なども確認する

#### B. ホームページ・LPの調査

`playwright-browser` スキルを使用して以下を実行する:

1. **ホームページのスクレイピング** — meta情報、見出し構造、主要コンテンツ、訴求ポイントを取得
2. **LPの特定と確認** — ユーザーが把握していないLPがないか調査する
3. **サービスサイトの構造確認** — 料金ページ、機能ページ、導入事例ページ等の有無と内容
4. **スクリーンショットの撮影** — ホームページとすべてのLPをフルページで撮影

```bash
# ホームページ
node $SKILL_DIR/../playwright-browser/scripts/browser.mjs screenshot <HP URL> --output <出力先>/hp-01.png --full-page

# LP
node $SKILL_DIR/../playwright-browser/scripts/browser.mjs screenshot <LP URL> --output <出力先>/lp-01.png --full-page
```

#### C. SNS・外部プレゼンスの調査

ユーザーが把握していないSNSアカウントや外部掲載がないか確認する:
- WebSearchで「[会社名] X(Twitter)」「[会社名] Instagram」等を検索
- 見つかったアカウントのフォロワー数、投稿頻度、直近の投稿傾向を確認する
- レビューサイト、メディア掲載、プレスリリースの有無も確認する

#### D. Google検索での可視性調査

`seo-analysis` スキルの「検索順位の調査」フローに従い、主要キーワードでの検索順位を調査する。

#### E. サイトパフォーマンス調査（PageSpeed Insights）

`seo-analysis` スキルの [PageSpeed Insights リファレンス](../seo-analysis/references/pagespeed-insights.md) に従い、PSI データを取得する。

#### F. 広告運用の調査

`playwright-browser` スキルで以下の広告ライブラリにアクセスし、自社の広告出稿状況を調査する。

| ツール | URL | 用途 |
|--------|-----|------|
| Meta広告ライブラリ | `https://www.facebook.com/ads/library/` | Facebook/Instagram広告の検索 |
| Google広告の透明性センター | `https://adstransparency.google.com/?region=JP` | Google広告の検索 |
| TikTok Top Ads | `https://ads.tiktok.com/business/creativecenter/inspiration/topads/pc/ja?period=30&region=JP` | TikTok広告の確認 |

**調査手順:**
1. 会社名・ブランド名・サービス名で各ライブラリを検索する
2. 該当する広告が見つかった場合のみ、クリエイティブ内容、訴求ポイント、遷移先LPを記録する
3. 広告が見つかった場合のみ、広告クリエイティブとLPのスクリーンショットを撮影する（広告が見つからなかった場合はスクリーンショット不要）

```bash
# 広告クリエイティブ
node $SKILL_DIR/../playwright-browser/scripts/browser.mjs screenshot <広告ライブラリURL> --output <出力先>/ad-01.png

# 広告のLP
node $SKILL_DIR/../playwright-browser/scripts/browser.mjs screenshot <LP URL> --output <出力先>/ad-lp-01.png --full-page
```

#### G. ECモール調査（Amazon/楽天）

物販・EC事業の場合、自社商品のECモールでの現状を調査する。

**Amazon:**
- `playwright-browser` で `https://www.amazon.co.jp/s?k=[ブランド名]` にアクセス
- 自社商品の検索順位、BSR（ベストセラーランキング）、レビュー数・平均評価、価格、A+コンテンツ有無を記録
- スポンサー広告の出稿状況（自社商品がスポンサー枠に表示されているか）

**楽天市場:**
- `playwright-browser` で `https://search.rakuten.co.jp/search/mall/[ブランド名]/` にアクセス
- 自社店舗・商品の検索順位、レビュー数・平均評価、価格、楽天ランキング掲載状況を記録
- RPP広告の出稿状況

#### H. アプリストア調査

アプリ事業の場合、自社アプリのストアでの現状を調査する。

- `playwright-browser` で App Store / Google Play の自社アプリページにアクセス
- カテゴリランキング順位、平均評価、レビュー数、DL数（Google Playは表示あり）を記録
- アプリ名・サブタイトル・説明文・スクリーンショット構成を確認
- 低評価レビューの主な不満点を把握

#### I. 口コミ・レビューサイト調査

レビューが購買意思決定に影響する業種で実施する。`competitor-analysis` の [調査ガイド](../competitor-analysis/references/research-guide.md) のセクション8「口コミ・レビューサイト調査」にある業種別レビューサイト一覧を参照し、該当するサイトで自社の評価・口コミを調査する。

- 総合評価（★）、レビュー件数、評価の分布
- 好評レビューの共通点（自社の強み）
- 低評価レビューの共通点（改善すべき点）
- 返信対応の状況

### エッジケース対応

| ケース | 対応 |
|--------|------|
| **新規事業（ローンチ前）** | Web調査はほぼスキップ。Phase 1 のヒアリングを充実させ、計画段階の情報（想定ターゲット、想定ビジネスモデル等）を記録する |
| **個人事業・フリーランス** | 「従業員数」「資本金」「資金調達」は省略可。屋号・ブランド名を中心に記録する |
| **複数サービスを持つ企業** | 企業全体のプロファイルを1つ作成し、「事業内容」セクション内でサービスごとにサブセクションを設ける |
| **外部企業を調べたい場合** | このスキルは自社・自プロジェクト専用。外部企業の調査は `competitor-analysis` を案内する |

### Phase 3: ドキュメント化

収集した全情報をマークダウンファイルとしてまとめる。情報が得られなかった項目は「未確認」と記載し、空欄にしない。実施しなかった調査項目のセクションはテンプレートから削除する。

## アウトプットルール

### 包括的かつ事実ベースの記述

- ユーザーから聞いた情報とWeb調査で得た情報を統合してまとめる
- 分析や評価は含めない（それは後続のスキルの役割）。事実のみを記録する
- URLは必ず記載する
- 情報が得られなかった項目は「未確認」と明記する

### ファイル出力

出力先はプロジェクトのディレクトリ構造（CLAUDE.md）に従う。スクリーンショット・PSI生データはコロケーション原則に従い、プロファイル本体と同じディレクトリに配置する。

### 出力テンプレート

[references/output-template.md](references/output-template.md) を参照。Phase 3 でドキュメント化する際にこのテンプレートをベースに使用する。

## 関連スキルとの連携

| スキル | 連携内容 |
|--------|----------|
| **industry-landscape** | このプロファイルで把握した業界・事業領域を起点に、業界の前提知識を調査する |
| **market-analysis** | ターゲット顧客・展開地域の情報をもとに、市場規模・顧客像を深掘りする |
| **competitor-analysis** | 事業内容・ポジショニングをもとに、直接・間接競合を特定・分析する |
| **regulatory-research** | 事業領域・展開地域をもとに、関連法規制・許認可・公的支援を調査する |
| **seo-analysis** | Phase 2 の検索可視性・PSI調査で連携。プロファイル作成後の本格SEO分析にも活用 |
| **playwright-browser** | Phase 2 のHP/LP調査、SNS確認、広告ライブラリ調査でブラウザ操作に使用する |

このスキルは他のリサーチスキルの前段階として最も効果を発揮する。プロファイルを先に作成しておくと、後続スキルが自社の文脈を参照でき、調査の精度と効率が上がる。
