# 競合分析（Competitor Analysis）

直接・間接の競合を特定し、各社のサービス内容・強み・弱み・ポジショニング・広告戦略を調査してレポートにまとめる。

## 前提条件

以下のスキルが完了していることが望ましい:
- `industry-landscape` — 対象業界の前提知識調査
- `market-analysis` — 市場規模・市場構造・価格帯・顧客像の調査

また `business-profile` が完了している場合、自社の現状（デジタルプレゼンス・広告状況・事業フェーズ等）を参照でき、比較の精度が上がる。

## ワークフロー

```
Phase 1: ヒアリング → Phase 2: 競合の特定 → Phase 3: 調査実施 → Phase 4: 分析・レポート化
```

---

### Phase 1: ヒアリング

ユーザーに段階的に質問し、調査の方向性を定める。

#### Step 1: 基本情報の確認

- 自社のサービス・商品の概要（`profile` があればそこから引き継ぐ）
- ユーザーが既に把握している競合名（あれば）
- 競合分析の目的（差別化戦略の立案 / 広告戦略の参考 / 価格設定の参考 / 市場参入の判断 等）

#### Step 2: 調査範囲の確認

- ビジネスの種別（SaaS / EC・物販 / アプリ / ローカルビジネス / その他）
- ターゲット市場（国内 / 海外 / 両方）
- 特に気になっている観点（価格 / 広告 / SNS / 商品ページ / レビュー 等）

### Phase 2: 競合の特定

ヒアリングで得た情報をもとに、直接競合・間接競合を特定する。

1. ユーザーが挙げた競合を起点に、WebSearch で追加の競合を探す
2. 直接競合 3〜5社、間接競合 2〜3社をリストアップする
3. リストをユーザーに提示し、**調査対象を確定する**（追加・削除の調整）

競合の特定方法の詳細は [references/research-guide.md](references/research-guide.md) のセクション1を参照。

### Phase 3: 調査実施

#### 調査メニューの提示

以下の調査項目をユーザーに提示し、実施する範囲を決める。ビジネスの状況に応じて推奨セットを提案する。

| # | 調査項目 | 推奨する場面 |
|---|---------|------------|
| A | 競合企業の基本情報 | 常に |
| B | Google検索での競合比較 | SEOが重要な集客チャネルの場合 |
| C | サイトパフォーマンス比較（PSI） | 自社サイトとの品質比較をしたい場合 |
| D | 広告運用の調査 | 広告戦略の参考にしたい場合 |
| E | SNS・ソーシャルメディア比較 | SNS運用の参考にしたい場合 |
| F | ECモール競合調査（Amazon/楽天） | 物販・EC事業の場合 |
| G | アプリストア競合調査 | アプリ事業の場合 |
| H | 口コミ・レビューサイト調査 | レビューが購買意思決定に影響する業種の場合 |

**ビジネス状況別の推奨:**
- **SaaS・Webサービス**: A〜E すべて推奨
- **EC・物販（自社EC中心）**: A〜E + 該当する場合 H
- **EC・物販（モール出品あり）**: A〜F + 該当する場合 H
- **アプリ事業**: A〜E + G
- **ローカルビジネス**: A + D + E + H
- **新規参入の市場調査**: A + B + D（まず概要を把握）

#### 各調査項目の実施

各項目の具体的な調査手順・クエリ例・確認ポイントは [references/research-guide.md](references/research-guide.md) を参照すること。

##### A. 競合企業の基本情報

各競合について売上規模・従業員数・事業内容・代表者・URL等を調査する。research-guide セクション2を参照。

##### B. Google検索での競合比較

`seo-analysis` スキルの「検索順位の調査」フローに従い、主要キーワードでの検索順位を自社・各競合で比較する。research-guide セクション3を参照。

##### C. サイトパフォーマンス比較（PSI）

`seo-analysis` スキルの [PageSpeed Insights リファレンス](../seo-analysis/references/pagespeed-insights.md) に従い、自社と各競合の PSI データを取得・比較する。

##### D. 広告運用の調査

`playwright-browser` スキルで広告ライブラリにアクセスし、各競合の広告出稿状況を調査する。research-guide セクション4を参照。

| ツール | URL | 用途 |
|--------|-----|------|
| Meta広告ライブラリ | `https://www.facebook.com/ads/library/` | Facebook/Instagram広告の検索 |
| Google広告の透明性センター | `https://adstransparency.google.com/?region=JP` | Google広告の検索 |
| TikTok Top Ads | `https://ads.tiktok.com/business/creativecenter/inspiration/topads/pc/ja?period=30&region=JP` | TikTok広告の確認 |
| TikTokキーワードインサイト | `https://ads.tiktok.com/business/creativecenter/keyword-insights/pc/ja` | キーワードトレンド |

広告が見つかった場合のみ、`playwright-browser` でスクリーンショットを撮影する。

##### E. SNS・ソーシャルメディア比較

各競合のSNSアカウントを調査し、フォロワー規模・コンテンツ戦略・エンゲージメントを比較する。research-guide セクション5を参照。

##### F. ECモール競合調査（Amazon/楽天）

物販・EC事業の場合に実施する。`playwright-browser` でAmazon・楽天市場にアクセスし、検索順位・BSR・レビュー・価格帯・広告出稿を調査する。research-guide セクション6を参照。

##### G. アプリストア競合調査

アプリ事業の場合に実施する。App Store / Google Play での順位・評価・レビュー・ASO状況を調査する。research-guide セクション7を参照。

##### H. 口コミ・レビューサイト調査

レビューが購買意思決定に影響する業種で実施する。業種別の主要レビューサイトは research-guide セクション8を参照。

### 検索ルール

#### 複数ソースの原則

1つのトピックにつき**最低3〜5回**の異なるクエリで検索し、**2〜3件以上**のソースをWebFetchで本文まで読むこと。

#### クエリ設計

- **広→狭のファネル戦略**: まず全体像を掴み、次に絞り込む
- **毎回クエリを変える**: 類義語・視点を変える
- **年を入れる**: 鮮度が重要な情報には必ず年を明記
- **言語はビジネスに合わせる**: ターゲット市場・展開地域に応じた言語で検索する
- **比較・対立軸を使う**: 比較クエリは質の高い分析記事がヒットしやすい

#### 情報収集ツール

```
Step 1: WebSearchで検索 → URLリストを取得
Step 2: WebFetchまたはplaywright-browserのscrapeで本文を取得
```

### Phase 4: 分析・レポート化

調査結果を統合し、レポートとしてまとめる。

1. **比較表の作成** — 各調査項目について競合横断の比較表を作成する
2. **示唆の導出** — 競合から学べる点、差別化の機会を抽出する
3. **レポートの出力** — [references/output-template.md](references/output-template.md) をベースに使用する

### エッジケース対応

| ケース | 対応 |
|--------|------|
| **競合が多すぎる（10社以上）** | 直接競合の中から売上規模・ターゲットが最も近い5社に絞る。残りは「その他の競合」として名前・URLのみリスト化 |
| **競合がほぼいない（ニッチ市場）** | 間接競合（代替手段）を中心に調査する。顧客が現在どのように課題を解決しているかに焦点を当てる |
| **新規事業で自社プロダクトがまだない** | 検索順位・PSI等の自社比較はスキップ。競合の強み・弱みの把握と参入機会の特定に集中する |
| **海外競合と国内競合が混在** | 国内・海外で分けてリストアップし、比較表も分離する。為替や市場の違いを明記する |
| **非公開企業で情報が取れない** | 推定値を使用し「推定」と明記する。求人情報、プレスリリース、SNS、レビューサイトから間接的に情報を収集する |
| **自社情報がまだ整理されていない** | `business-profile` スキルを先に実行するよう案内する |

## アウトプットルール

### 網羅的な記述

- 各競合のサービス内容・料金・強み・弱みは具体的に記載する
- 複数ソースから得た情報はすべて記載し、差異があればそれも明記する
- 「など」「その他多数」で省略せず、把握した項目はすべて列挙する
- 広告ライブラリで確認した内容も具体的に記述する

### ソースURLの記載

すべての発見事項・データには、情報源のURLを必ず記載すること。URLのないデータは出力に含めない。

### スクリーンショット

広告が見つかった場合のみ、`playwright-browser` でスクリーンショットを撮影する。撮影対象:
- 広告ライブラリで見つけた主要な広告クリエイティブ
- 広告から遷移するランディングページ（フルページ）
- 競合のホームページ

マークダウンレポート内では、スクリーンショットのファイルパスを記載し、どの画像が何を撮影したものかを明記する。

### ファイル出力

出力先はプロジェクトのディレクトリ構造（CLAUDE.md）に従う。スクリーンショット・PSI生データはコロケーション原則に従い、レポート本体と同じディレクトリに配置する。

**ファイル構成:** 競合ごとにファイルを分割する。メインファイルには要約・競合リスト・横断比較表・示唆を記載し、各社の詳細調査は個別ファイルに分ける。

```
competitor-analysis.md              # メイン（要約・競合リスト・横断比較・示唆）
competitor-analysis/
├── [競合名1].md                    # 競合1の詳細（基本情報・広告・SNS等）
├── [競合名2].md                    # 競合2の詳細
├── [競合名3].md                    # 競合3の詳細
├── psi-[競合名]-mobile.json        # PSI生データ
└── [競合名]-ad-01.png              # スクリーンショット
```

メインファイルから各社の詳細ファイルへ相対リンクで参照する。

### 出力テンプレート

[references/output-template.md](references/output-template.md) を参照。Phase 4 でレポート化する際にこのテンプレートをベースに使用する。

## 関連スキルとの連携

| スキル | 連携内容 |
|--------|----------|
| **business-profile** | 自社の現状（デジタルプレゼンス・広告状況等）を把握し、競合との比較基準にする |
| **industry-landscape** | 業界構造・バリューチェーンの理解をもとに、競合の位置づけを正確に把握する |
| **market-analysis** | 市場規模・顧客像の情報をもとに、競合のターゲット・シェアを推定する |
| **seo-analysis** | 検索順位の調査（Phase 3-B）とPSI比較（Phase 3-C）で使用する |
| **playwright-browser** | 広告ライブラリ・ECモール・SNS・アプリストアのブラウザ操作に使用する |

このスキルの結果は、後続の広告戦略スキルやCROスキルの入力として活用できる。競合の強み・弱みを把握した上で自社の差別化戦略を立てるのが典型的な流れ。
