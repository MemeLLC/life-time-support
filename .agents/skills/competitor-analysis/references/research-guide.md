各項目で最低2〜3回の異なるクエリで検索すること。検索言語はビジネスのターゲット市場・展開地域に合わせる。

## 1. 競合の特定

**調べること:**
- 直接競合（同じ課題を同じ方法で解決するサービス・企業）
- 間接競合（同じ課題を別の方法で解決するサービス・企業）
- 直接競合3〜5社、間接競合2〜3社をリストアップする

**クエリ例:**
- `"[サービス名] 競合"`
- `"[サービス名] alternatives"`
- `"[業種] 比較 ランキング"`
- `"[業種] カオスマップ"`
- `"[サービス名] vs"`

## 2. 競合企業の基本情報

各競合について以下の情報を調査する。

**調べること:**
- 売上規模（年商、売上高）
- 従業員数
- 展開しているビジネスの内容（サービス概要、主力商品、ターゲット層）
- 代表者の名前
- ホームページのURL
- LP（ランディングページ）のURL

**クエリ例:**
- `"[競合名] 会社概要"`
- `"[競合名] 売上 従業員数"`
- `"[競合名] 代表"`
- `"[競合名] 企業情報"`
- `site:[競合ドメイン] company` / `site:[競合ドメイン] about`

**情報源の優先順:**
- 競合の公式サイト（会社概要ページ、IR情報）
- 企業データベース（マイナビ、Wantedly、INITIAL、Crunchbase など）
- 上場企業の場合はIR資料（有価証券報告書、決算資料）
- プレスリリース（PR TIMES など）

**注意:**
- 各社の公式サイトをWebFetchで確認し、会社概要ページから基本情報を取得する
- 非公開企業の場合、売上規模は推定値でも可（推定であることを明記する）

## 3. Google検索での競合比較

`playwright-browser` スキルを使い、主要キーワードでのGoogle検索結果を取得して自社と競合の順位を比較する。

**調べること:**
- 主要キーワードでの自社・各競合の検索順位
- 各キーワードの検索結果上位を占めているサイトの傾向
- 広告枠での自社・競合の出稿状況
- リッチリザルト（FAQ、レビュー等）の表示有無

**キーワード選定:**

業界・サービスカテゴリから5〜10個を選定する。`business-profile` が完了している場合はそこから引き継ぐ。

| カテゴリ | クエリ例 |
|---------|---------|
| サービスカテゴリ | `[業種] サービス`, `[提供内容] ツール` |
| 比較・検討系 | `[業種] 比較`, `[サービスカテゴリ] おすすめ` |
| 課題・ニーズ系 | `[顧客の課題] 解決`, `[ニーズ] 方法` |
| 地域系（該当する場合） | `[地域名] [サービスカテゴリ]` |

**調査手順:**

各キーワードについてテキスト結果の取得とスクリーンショット撮影を行う。

```bash
# テキスト結果の取得（上位10件）
node $SKILL_DIR/../playwright-browser/scripts/browser.mjs search "[キーワード]" --results 10

# 検索結果画面のスクリーンショット
node $SKILL_DIR/../playwright-browser/scripts/browser.mjs interact "https://www.google.com/search?q=[キーワード]&hl=ja&gl=jp" \
  --actions '[{"action":"wait","timeout":2000},{"action":"screenshot","path":"research/screenshots/competitor-analysis/serp-[keyword-slug].png"}]'
```

**結果の記録:**
- キーワードごとに自社・各競合の順位を一覧テーブルにまとめる
- 1ページ目（上位10件）に入っていない場合は「圏外」と記載
- 広告枠に自社・競合が出稿しているかも記録する

## 4. 広告運用の調査

`playwright-browser` スキルで広告ライブラリにアクセスし、各競合の広告運用状況を調査する。

**調べること:**
- 各競合がどのようなLPを出しているか（LP URL、構成、訴求ポイント）
- 広告クリエイティブの内容（コピー、ビジュアル、CTA）
- 出稿プラットフォーム（Meta、Google、TikTok など）
- 広告の出稿量・掲載期間

**調査ツール:**
- Meta広告ライブラリ — 各競合名で検索
- Google広告の透明性センター — 各競合名で検索
- TikTok Top Ads — 業界関連の広告パフォーマンス上位を確認
- TikTokキーワードインサイト — 競合が使用しているキーワードを確認

**スクリーンショットの撮影:**

広告クリエイティブとLPは `playwright-browser` スキルの screenshot コマンドで撮影する。

```bash
# 広告ライブラリの検索結果画面
node .agents/skills/playwright-browser/scripts/cli.mjs screenshot <広告ライブラリURL> --output research/screenshots/competitor-analysis/[競合名]-ad-01.png

# 競合のLP（ページ全体）
node .agents/skills/playwright-browser/scripts/cli.mjs screenshot <LP URL> --output research/screenshots/competitor-analysis/[競合名]-lp-01.png --full-page

# 競合のホームページ
node .agents/skills/playwright-browser/scripts/cli.mjs screenshot <HP URL> --output research/screenshots/competitor-analysis/[競合名]-hp-01.png --full-page
```

**注意:**
- 広告のURL、LP URLは必ず記録する
- スクリーンショットは `research/screenshots/competitor-analysis/` に保存する
- マークダウンレポート内でスクリーンショットのパスを記載し、どの画像が何を撮影したものか明記する

## 5. SNS・ソーシャルメディア比較

競合のSNSアカウントを調査し、フォロワー規模・コンテンツ戦略・エンゲージメントを比較する。

**調べること:**
- 各プラットフォームのフォロワー数
- 投稿頻度（週あたりの投稿数）
- エンゲージメント傾向（いいね・コメント・シェアの多い投稿の特徴）
- コンテンツの種類（画像/動画/テキスト、教育系/プロモ系/UGC活用 等）
- プロフィールのCTA・リンク先

**対象プラットフォーム:**

| プラットフォーム | 確認方法 |
|----------------|---------|
| X（旧Twitter） | `playwright-browser` で `https://x.com/[アカウント名]` にアクセス |
| Instagram | `playwright-browser` で `https://www.instagram.com/[アカウント名]/` にアクセス |
| TikTok | `playwright-browser` で `https://www.tiktok.com/@[アカウント名]` にアクセス |
| YouTube | `playwright-browser` で企業チャンネルにアクセス |
| LINE公式 | 競合サイト上のLINE追加リンクから確認（友だち数は非公開の場合が多い） |

**クエリ例:**
- `"[競合名] Twitter"` / `"[競合名] Instagram"`
- `"[競合名] 公式アカウント"`
- `"[競合名] SNS"`

**注意:**
- アカウントが見つからない場合は「未運用」と記録する
- フォロワー数は概数（1.2万 等）で記録して問題ない

## 6. ECモール競合調査

物販・EC事業の場合に実施する。Amazon と楽天市場で競合の商品ページ・価格・レビュー・広告出稿を調査する。

### Amazon

**調べること:**
- 主要キーワードでの検索結果上位の商品（誰が上位を占めているか）
- 競合のBSR（ベストセラーランキング）とカテゴリ順位
- レビュー数・平均評価・低評価レビューの主な不満点
- 価格帯・クーポン/ポイント施策
- 商品ページの構成品質（タイトル・画像枚数・A+コンテンツ有無・動画有無）
- スポンサー広告の出稿状況（検索結果でスポンサー枠に表示されているか）
- FBA/FBM の判別（Prime マーク有無）

**調査手順:**
1. `playwright-browser` で `https://www.amazon.co.jp/s?k=[主要キーワード]` にアクセス
2. 検索結果の上位10商品をリストアップし、スクリーンショットを撮影
3. 主要競合の商品ページに移動し、詳細情報を記録
4. 「スポンサー」表示のある商品を記録（広告出稿の証拠）

**クエリ例（Web検索で補足情報を得る場合）:**
- `"[競合ブランド名] Amazon"`
- `"[商品カテゴリ] Amazon ランキング"`
- `"[商品カテゴリ] Amazon 売れ筋"`

### 楽天市場

**調べること:**
- 主要キーワードでの検索結果上位の商品・店舗
- レビュー数・平均評価
- 価格帯・ポイント倍率・クーポン施策
- 商品ページの構成品質（画像・説明文の充実度）
- 楽天ランキング（デイリー/週間）への掲載状況
- RPP広告の出稿状況（検索結果で「PR」表示されているか）

**調査手順:**
1. `playwright-browser` で `https://search.rakuten.co.jp/search/mall/[主要キーワード]/` にアクセス
2. 検索結果の上位10商品をリストアップし、スクリーンショットを撮影
3. 楽天ランキング `https://ranking.rakuten.co.jp/` で該当カテゴリを確認

### ECモール共通の注意点

- 価格比較は同一商品・同一容量で揃えること（単価計算が必要な場合あり）
- レビューの「低評価（★1-2）」は競合の弱点発見に有用。主な不満点を3つ程度抽出する
- セール時期（Prime Day、楽天スーパーSALE等）は通常時と価格が異なるため、調査日を明記する

## 7. アプリストア競合調査

アプリ事業の場合に実施する。App Store と Google Play での競合アプリの順位・評価・ASO状況を調査する。

**調べること:**
- カテゴリランキング順位
- 平均評価・レビュー数
- DL数（Google Playは表示あり、App Storeは推定）
- アプリ名・サブタイトル・説明文のキーワード（ASO）
- スクリーンショット・プレビュー動画の構成
- 最終更新日（メンテナンス頻度の指標）
- 低評価レビューの主な不満点

**調査手順:**
1. `playwright-browser` で各ストアの検索結果にアクセス
   - App Store: `https://apps.apple.com/jp/search?term=[キーワード]`
   - Google Play: `https://play.google.com/store/search?q=[キーワード]&c=apps&hl=ja`
2. 上位アプリのページにアクセスし、詳細情報を記録
3. カテゴリランキングも確認する

**クエリ例（Web検索で補足情報を得る場合）:**
- `"[競合アプリ名] ダウンロード数"`
- `"[アプリカテゴリ] アプリ 比較 ランキング"`

## 8. 口コミ・レビューサイト調査

レビューが購買意思決定に影響する業種で実施する。業種に応じたレビューサイトで競合の評価・口コミ傾向を調査する。

**業種別の主要レビューサイト:**

| 業種 | レビューサイト |
|------|-------------|
| SaaS・ビジネスツール | G2、Capterra、ITreview、BoxIL |
| 飲食 | 食べログ、ホットペッパーグルメ、Retty、Google Maps |
| 美容・サロン | ホットペッパービューティー、Google Maps |
| 旅行・宿泊 | じゃらん、楽天トラベル、Booking.com、Google Maps |
| 医療 | Google Maps、病院口コミ検索サイト |
| 教育・スクール | Google Maps、各比較サイト |
| EC・物販 | Amazon レビュー、楽天レビュー（ECモール調査と統合） |
| ローカルビジネス全般 | Google Maps（Google ビジネスプロフィール） |

**調べること:**
- 総合評価（★）と評価の分布
- レビュー件数
- 好評レビューの共通点（競合の強み）
- 低評価レビューの共通点（競合の弱点 = 自社の差別化機会）
- 返信対応の有無・質（顧客対応姿勢の指標）

**クエリ例:**
- `"[競合名] 口コミ"`
- `"[競合名] 評判"`
- `"[競合名] レビュー"`
- `site:[レビューサイト] [競合名]`

**注意:**
- レビューは件数だけでなく内容の傾向を把握する。「安い」「対応が良い」等のキーワード頻度を意識する
- Google Mapsのレビューは `playwright-browser` でアクセスし、評価・件数・主要口コミを記録する
