# 法規制・公的支援調査

Web検索（WebSearch / WebFetch）を活用し、対象事業に関連する法規制・許認可・公的支援を調査する。

## 前提条件

以下のスキルが完了していることが望ましい:
- `industry-landscape` — 対象業界の前提知識調査
- `market-analysis` — 市場規模・市場構造・価格帯・顧客像の調査
- `competitor-analysis` — 競合分析

## 調査項目

法規制・公的支援の詳細な調査項目・クエリ例は [references/regulatory-research.md](references/regulatory-research.md) を参照すること。

## 検索ルール

### 複数ソースの原則

1つのトピックにつき**最低3〜5回**の異なるクエリで検索し、**2〜3件以上**のソースをWebFetchで本文まで読むこと。1つのサイトの情報だけでアウトプットしてはならない。

### クエリ設計

- **広→狭のファネル戦略**: まず全体像を掴み、次に絞り込む
- **毎回クエリを変える**: 同じ言い回しの繰り返しは同じ結果しか返さない。類義語・視点を変える
- **年を入れる**: 鮮度が重要な情報には必ず年を明記する
- **言語はビジネスに合わせる**: ターゲット市場・展開地域に応じた言語で検索する（日本ローカルなら日本語、海外展開なら英語や現地語など）

### 検索演算子の活用

検索精度を上げるために以下の演算子を積極的に使うこと：

| 演算子 | 用途 | 例 |
|--------|------|-----|
| `"完全一致"` | フレーズ完全一致検索 | `"電子商取引 法規制"` |
| `site:` | 特定サイト内検索 | `site:meti.go.jp 電子商取引` |
| `filetype:` | ファイル形式指定 | `filetype:pdf [業種] 法規制 ガイドライン` |

**一次情報を狙うsite:指定の例：**
- 官公庁: `site:go.jp`（経産省、総務省など）
- 法令データベース: `site:e-gov.go.jp`

### 検索→取得の2ステップ

```
Step 1: WebSearchで検索 → URLリストを取得
Step 2: WebFetchで上位の有望なURLの中身を読み込む → 詳細な情報を取得
```

スニペットだけで判断せず、重要な情報はWebFetchで本文を読むこと。

## アウトプットルール

### 網羅的な記述

- 法令名・条文番号は正確に記載する
- 複数ソースから得た情報はすべて記載し、差異があればそれも明記する
- 「など」「その他多数」で省略せず、把握した項目はすべて列挙する

### ソースURLの記載

すべての発見事項・データには、情報源のURLを必ず記載すること。URLのないデータは信頼性が担保できないため、出力に含めてはならない。

### ファイル出力

リサーチ結果は必ずマークダウンファイルとして出力する。出力先はユーザーの指示に従う。

[references/output-template.md](references/output-template.md) を参照。

## 関連スキルとの連携

| スキル | 連携内容 |
|--------|----------|
| **industry-landscape** | 業界の前提知識（用語・構造・規制環境）が揃っていると、法規制調査の検索精度が上がる |
| **market-analysis** | 市場の参入障壁に規制要因がある場合、法規制調査で詳細を把握する |
| **competitor-analysis** | 競合の許認可取得状況やコンプライアンス対応を把握する際の参考になる |
| **business-profile** | 自社の事業内容が整理済みであれば、必要な許認可・規制の特定が正確になる |
| **playwright-browser** | 官公庁のWebサイト、法令データベース、補助金ポータルの情報取得に使用する |
